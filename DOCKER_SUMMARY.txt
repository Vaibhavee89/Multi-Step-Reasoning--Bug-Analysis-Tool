⚡ QUICK COMMANDS:

Start:
  ./docker-start.sh prod         # Production mode
  ./docker-start.sh dev          # Development mode

Stop:
  ./docker-start.sh stop         # Stop all services

Logs:
  ./docker-start.sh logs         # View all logs
  docker-compose logs -f backend # Backend only
  docker-compose logs -f frontend # Frontend only

Status:
  docker-compose ps              # Service status
  docker stats                   # Resource usage

Restart:
  docker-compose restart         # Restart all
  docker-compose restart backend # Restart backend

Rebuild:
  docker-compose up --build -d   # Rebuild and start

Clean:
  docker-compose down            # Stop and remove containers
  docker-compose down -v         # Also remove volumes
  docker system prune -a         # Clean everything

Shell:
  docker exec -it bug-analysis-backend bash    # Backend shell
  docker exec -it bug-analysis-frontend sh     # Frontend shell

🐛 TROUBLESHOOTING:

Can't start containers?
  → Check Docker is running: docker ps
  → Check ports not in use: lsof -i :80 -i :8000
  → Check .env file exists with API key

Backend won't start?
  → View logs: docker-compose logs backend
  → Check API key in .env
  → Verify network: docker network ls

Frontend can't reach backend?
  → Check both containers running: docker-compose ps
  → Verify network bridge: docker network inspect bug-analysis-network
  → Check CORS settings in backend/main.py

Build failures?
  → Clean build: docker-compose down && docker system prune -a
  → Rebuild: docker-compose up --build
  → Check Docker disk space: docker system df

Performance issues?
  → Check resources: docker stats
  → Increase Docker memory limits
  → Use production mode for better performance
