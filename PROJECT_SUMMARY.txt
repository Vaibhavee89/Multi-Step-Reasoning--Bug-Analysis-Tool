🎯 KEY FEATURES:

1. Multi-Step Reasoning (ReAct Pattern)
   ───────────────────────────────────
   Thought → Action → Observation → Repeat

   The agent doesn't just run tools - it THINKS:
   - Plans the analysis approach
   - Reasons about findings
   - Combines information from multiple sources
   - Validates conclusions

2. Autonomous Tool Selection
   ─────────────────────────
   Agent automatically chooses which tools to use based on:
   - The analysis task
   - Previous observations
   - Reasoning about what info is needed

3. Comprehensive Bug Detection
   ───────────────────────────
   Finds:
   - Logical errors & runtime bugs
   - Security vulnerabilities (SQL injection, XSS, etc.)
   - Code quality issues
   - Performance bottlenecks
   - Anti-patterns & code smells

4. Transparent Reasoning
   ─────────────────────
   See exactly how the agent thinks:
   - Step-by-step reasoning traces
   - Tool selection rationale
   - Intermediate findings
   - Final synthesis

🤝 COMMON INTERVIEW QUESTIONS & ANSWERS:

Q: "How does your agent work?"
A: "It uses the ReAct pattern - Reasoning and Acting. The agent thinks
   about what information it needs, selects appropriate tools, interprets
   results, and repeats until it has a complete analysis. This multi-step
   reasoning allows it to find issues that simple static analysis misses."

Q: "What makes this different from a linter?"
A: "Linters follow fixed rules. My agent reasons about the code - it can
   trace bugs across multiple files, understand context, and synthesize
   findings from multiple sources. It's like having an expert code reviewer
   who thinks through the analysis systematically."

Q: "What was the biggest challenge?"
A: "Designing prompts that guide the agent to thorough analysis while
   staying focused. Also, ensuring tools return structured output that the
   LLM can reason about effectively. The key was balancing autonomy with
   control."

Q: "How would you scale this?"
A: "Add caching for frequently analyzed code, implement async processing
   for parallel tool execution, use embeddings for semantic code search,
   and add incremental analysis for large codebases."
